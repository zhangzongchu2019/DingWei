# DingWei 会话命名强制规范 v1.0

> 来源:张宗楚 2026-07-09 派发任务正文转录(原文档位于其本地仓 content-center/toolkit/dingwei/docs/,本机不存在,以此转录为实现依据)。实现须严格按本规范。

## 一、命名格式(强制)

会话注册名 = `<owner_key>-<短名>-<key末4位>`

- 最终正则:`^[a-z0-9]+-[a-z0-9]+-[0-9a-f]{4}$`
- 例:`zzc-manager-2642`、`fulei-dev1013-3dd6`
- 语义校验:
  - `owner_key` 段必须等于该 key 实际绑定成员的 owner_key;
  - 末4位段必须等于 `SH_KEY_ID` 真实末4位。

## 二、用户体验(B 方案·自动拼装)

用户启动只填短名:

- `SH_SESSION_NAME=<短名>`(如 `manager`)
- `SH_OWNER=<owner_key>`(如 `zzc`)

客户端自动拼 `${SH_OWNER}-${SH_SESSION_NAME}-${SH_KEY_ID末4位}` 作为注册名。用户不手写 owner 段和 key 末位段。

## 三、客户端强制(Python,config.py 加载时)

文件:`tools/sessionhelper/sessionhelper/config.py`

1. 缺 `SH_OWNER` 或 `SH_SESSION_NAME` → 拒绝启动;
2. 短名非 `^[a-z0-9]+$`(含 `-`、空格、大写等)→ 拒绝启动,提示"短名只能小写字母数字,如 manager";
3. `SH_OWNER` 同规则校验;
4. 拼装后按最终正则复核。

任一步不过 → 直接 `SystemExit` 拒绝启动,打印正确格式 + 本机应有示例。**绝不带病连接。**

## 四、服务端强制(Go,internal/m8/hub.go 的 /ws/session 握手)

位置:key/secret 校验通过之后。

1. 注册名不匹配最终正则 → 400/401 拒绝连接,回执"会话名不合规,须为 `<owner_key>-<短名>-<key末4位>`,如 fulei-dev1013-3dd6";
2. 末4位段 ≠ keyID 末4位 → 拒;owner_key 段 ≠ 该 key 绑定成员 owner_key → 拒;
3. 全部通过才 `registerSessionEndpoint` + 入 `sessionClients`。

## 五、在线清单渲染修正(配套,必须一起改)

位置:`hub.go` `renderOnlineDirectory`(~L3020)、`shortSessionName`(~L3055)。

现用 `shortSessionName(full_session_name)` 派生寻址是 bug 源(导致 dev1013 被显示成 #bg)。改为:

- `#短名` / `@owner#短名` / view 地址一律基于真实 `ep.SessionName`;
- `shortSessionName` 升级:识别 `<owner>-<短名>-<末4>` 三段式,切出中段展示 `#短名`;
- `全名:` 展示完整注册名;
- 路由 / view 底层仍用完整注册名,保证唯一可达。

## 六、灰度落地(避免一刀切踢全员下线)

现所有会话命名都不规范(团队 zzc-* 缺 key 末位、符磊 dev1013、is3 poc-ops/ts3-dev、sec-ops)。落地顺序:

1. Hub 先只告警:扫出全部不合规清单(不拒);
2. 运维统一改各端 config(张宗楚已安排 ai-dev 运维一次性改);
3. 升级带校验的新客户端;
4. 全部合规后 Hub 切硬拒绝;
5. 现有脏 `full_session_name` 顺带清掉。

实现上需要一个灰度开关(建议 env,如 `WP_NAME_ENFORCE=off|warn|enforce`,默认 warn),warn 模式记录/输出不合规清单但放行。
